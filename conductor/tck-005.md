# TCK-005: Version Control Seraphina's Soul and Interface

## Background

We established a powerful "Burning One" persona for Seraphina using a custom `SOUL.md` file and a Dark Academia styled Canvas UI. However, this state lived ephemerally in the `~/.openclaw` directory, entirely outside of version control. If the host machine were lost, Seraphina's identity and aesthetic would be unrecoverable.

## Proposed Solution

Extract Seraphina's file system representation into the `openClaw/conductor/` repository structure. We will version-control her active files while utilizing symbolic links to ensure OpenClaw can still access her state dynamically.

## Implementation Steps

1. **Directory Structure:** Create `conductor/agents/seraphina/workspace` and `conductor/agents/seraphina/canvas`.
2. **State Relocation:** Move `~/.openclaw/agents/seraphina/workspace/SOUL.md` and `~/.openclaw/canvas/index.html` into the newly created conductor directories.
3. **Symbolic Linking:** Create symlinks from the original `~/.openclaw/` paths pointing directly to the version-controlled artifacts in the `openClaw/conductor/` repository.
4. **Git Tracking:** Force-add the newly extracted files to the Git index to immortalize Seraphina's identity in the repository history.
